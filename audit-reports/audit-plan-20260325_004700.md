## FASE 3A — Desalineación Planes (Detalle)

### PLAN-001: Backend tiene 4 planes de dealer, Frontend muestra 6
- **Detalle:** Backend (PlanConfiguration.cs): Libre($0), Visible($29), Pro($89), Elite($199). Frontend (dealer-plans-section.tsx): Libre, Visible(RD$1,682), STARTER(RD$3,422), PRO(RD$5,742), ÉLITE(RD$20,242), ENTERPRISE(RD$34,742). Planes STARTER y ENTERPRISE no existen en backend.
- **Impacto:** Si un dealer compra STARTER o ENTERPRISE, el backend no sabe qué plan aplicar
- **Acción:** Decidir: ¿agregar al backend o eliminar del frontend?

### PLAN-002: Precios INCONSISTENTES entre backend y frontend (Pro y Elite)
- **Detalle:** PRO: Backend $89/mes ≈ RD$5,385, Frontend muestra RD$5,742 (diferencia +RD$357). ELITE: Backend $199/mes ≈ RD$12,040, Frontend muestra RD$20,242 (diferencia +RD$8,202!)
- **Impacto:** Cliente paga precio de frontend pero backend registra precio diferente → discrepancia contable
- **Acción:** Sincronizar precios. Definir fuente de verdad (¿backend API o frontend config?)

### PLAN-003: Seller plans NO EXISTEN en backend
- **Detalle:** Frontend /vender muestra: Gratis, Premium(RD$579), PRO(RD$2,029). Frontend /cuenta/suscripcion muestra: Libre, Estándar($9.99/pub), Verificado($34.99/mes). Backend PlanConfiguration.cs solo tiene planes de DEALER. PlanFeatureLimits.cs no tiene seller-specific limits.
- **Impacto:** Sellers no pueden realmente pagar/upgradar su plan porque el backend no lo soporta
- **Acción:** Implementar SellerPlan en backend o unificar seller como subset de dealer plans

### PLAN-004: Dos páginas de pricing para sellers con planes DISTINTOS
- **Detalle:** /vender muestra: Gratis, Premium(RD$579/mes, 5 pubs, 30 fotos), PRO(RD$2,029/mes, 15 pubs, 50 fotos). /cuenta/suscripcion muestra: Libre, Estándar($9.99/publicación), Verificado($34.99/mes, 3 pubs, 12 fotos). Son planes completamente diferentes con nombres, precios y features distintos.
- **Impacto:** Confusión total del usuario. ¿Cuál es el plan real?
- **Acción:** Unificar seller plans en una sola definición consistente

### PLAN-005: ChatAgent limits difieren entre frontend y backend
- **Detalle:** ÉLITE: Frontend dice 5,000 conv/mes Web + 5,000 WhatsApp. Backend PlanFeatureLimits: ChatAgentMonthlyMessages = 2,000. PRO: Frontend dice 300 conv/mes. Backend: ChatAgentMonthlyMessages = 500.
- **Impacto:** Si se implementa enforcement en backend, dealers recibirán menos de lo prometido
- **Acción:** Alinear limits. Si se vende 5,000, backend debe soportar 5,000

### PLAN-006: Feature limits de imágenes inconsistentes
- **Detalle:** Backend PlanFeatureLimits: Libre=10, Visible=20, Pro=30, Elite=40. Frontend /dealers: Libre=5, Visible=10, Starter=12, Pro=15, Elite=20. Frontend /vender: Gratis=5 fotos, Premium=30, PRO=50.
- **Impacto:** Backend permite más fotos que lo que el frontend promete (o viceversa)
- **Acción:** Definir límites definitivos y sincronizar
